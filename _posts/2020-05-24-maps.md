One of its features is that it allows users to visualize street networks of practically anywhere in the world. ) incredibly helpful in understanding how to use this Python package.

The first thing you want to do is install OSMnx. 

Using conda (recommended by the author):

```
conda install -c conda-forge osmnx
```

Using pip:

```
pip install osmnx
```

Next, let’s import the package. I’m going to use Jupyter notebook to create my custom map.

```
import osmnx 
from IPython.display import Image
%matplotlib inline
```
The function that’s going to help us here is `osmnx.plot.plot_figure_ground`. The complete list of parameters is found [here](https://osmnx.readthedocs.io/en/stable/osmnx.html#module-osmnx.plot), but I’ll go through the fundamental ones here:
- The centre point can be defined using `address` or `point`. `point` takes in a latitude and longitude as a tuple.
- `dist` controls how many metres to extend north, east, south, and west from the centre point.
- The width of the lines can be specified through `default_width`. 
- `edge_color` and `bgcolor` allow you to change the colour of the streets and that of the background, respectively.

Here’s the code I used to create the map you saw earlier:

```
# Create a tuple with latitude and longitude of Berlin (centred around Mitte)
coordinate = (52.5200, 13.4050)

# Create the plot
osmnx.plot.plot_figure_ground(point=coordinate, dist=2500, default_width=1,
                              edge_color='#f1f1f1', bgcolor='#212121', 
                              filename='berlin', show=True, dpi=800)
```
The output:

![berlin]({{ site.baseurl }}/images/berlin.png)

By default, OSMnx plots drivable public streets. However, it’s worthwhile to mention that some cities, namely those with an extensive network of bike routes, can look different depending on which network type is shown. This can be changed using the `network_type` parameter.

You can just save the output and stop here or add some finishing touches using an image editor of your choice. Since I don’t have access to a top-of-the-line graphic design software, I used Google Slides to add the border and text. And there you have it! You could definitely print these at home but be mindful of the image resolution (controlled with the `dpi` parameter) if you choose to do so.

Comparing Mapiful’s product to my MacGyvered implementation, there are a few noticeable differences. I’d actually argue that it’s easier to customize the map using OSMnx. First, you have more flexibility with regards to the amount of area covered by the map. When placing an order on Mapiful, you can either zoom in or out to decide how much or little area you want to include. However, it’s impossible to make fine adjustments. On the other hand, using OSMnx, you can specify precisely how many metres to extend north, east, south, and west from the centre point. Second, as I mentioned earlier, OSMnx lets you pick which type of network to plot.

However, the function I used to create the map only handles two different colours: one for the network and the other for the background. As a result, land and water are shown in the same colour. In the map of Berlin, you can actually see that there’s a gap that runs diagonally across the centre of the image. That’s where the Spree should be. This might not be an issue if you’re looking to visualize a landlocked city, but it might be inconvenient if you’re trying to create a map of a city on the water or a place with a dense canal network.
